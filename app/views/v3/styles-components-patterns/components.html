Components

Save time and help users with these accessible UI parts.

What are you trying to achieve?
  • Make a basic page
  • Make a form
  • Give clinical advice
  • Direct people to a service
  • Highlight important information

Action link

Help users get to the next stage of a journey quickly by signposting the start of a digital service.

Back link

Help users go back to the previous page in a multi-page transaction.

Breadcrumbs

Help users understand where they are in the website.

Buttons

Help users carry out an action on a page like starting an application or saving their progress.

Care cards

Help users identify and understand the sort of care they need, who to contact and how quickly.

Checkboxes

Let users select one or more options on a form.

Contents list

Allow users to navigate between related pages, for example about a single condition.

Date input

Help users enter a memorable date, like their date of birth.

Details

Make a page easier to scan by letting users reveal more detailed information only if they need it.

Do and Don't lists

Help users understand more easily what they should and shouldn't do.

Error message

Explain what went wrong and how to fix it.

Error summary

Summarise any mistakes a user has made.

Expander

Make a complex topic easier to digest by letting users reveal more detailed information only if they need it.

Fieldset

Group related form inputs.

Footer

Show users they are on an NHS service and to help them find links they expect at the bottom of our pages.

Header

Show users they are on an NHS service and help them get started in finding what they need.

Hint text

Use hint text to help users understand a question.

Images

Use images only if there is a real user need. Avoid unnecessary decoration.

Inset text

Help users identify and understand important content on the page, even if they don't read the whole page.

Pagination

Allow users to navigate between related pages, for example about a single condition.

Radios

Help users to select only 1 option from a list.

Review date

Reassure users that our information is up-to-date.

Select

Let users choose an option from a long list but only use it as a last resort.

Skip link

Help keyboard-only users skip to the main content on a page.

Table

Make it easier for users to compare and scan information.

Text input

Let users enter a single line of text.

Textarea

Let users enter more than 1 line of text.

Warning callout

Help users understand warning content on the page, even if they don't read the whole page.
